# Практика 5. Корзина и разграничение прав доступа

## Цель

Создать функциональную корзину для товаров, оформление заказов из корзины, авторизацию, регистрацию, выход из профиля и разграничение прав доступа.

## Выполненные шаги

### 1. Хранение корзины через сессии

Корзина реализована через Django sessions. Данные хранятся в `request.session` по ключу `cart` в формате:

```python
{
    "product_id": quantity
}
```

Основная логика находится в `bakery/views.py`:

- `CART_SESSION_KEY = 'cart'`;
- `_get_cart()` получает или создает корзину в сессии;
- `AddToCartView` добавляет товар;
- `UpdateCartItemView` изменяет количество;
- `RemoveFromCartView` удаляет товар;
- `CartView` собирает товары из сессии и считает итоговую стоимость.

В шаблонах добавлены формы добавления товаров в корзину:

- `bakery/templates/bakery/product_list.html`;
- `bakery/templates/bakery/product_detail.html`.

Страница корзины обновлена в `bakery/templates/bakery/cart.html`.

### 2. Создание заказа из корзины

В корзине добавлена кнопка `Создать заказ`. Для гостя вместо нее отображается ссылка на авторизацию.

Создание заказа реализовано в `CreateOrderView`:

- оформление доступно только авторизованным пользователям;
- заказ создается в транзакции;
- к заказу добавляются позиции `OrderItem`;
- итоговая сумма считается по актуальным ценам товаров;
- после создания заказа корзина очищается.

### 3. Привязка заказа к пользователю

В модель `Order` добавлено поле:

```python
user = models.ForeignKey(settings.AUTH_USER_MODEL, ...)
```

При создании заказа выполняется:

```python
Order.objects.create(user=request.user, total_price=total_price)
```

Для изменения структуры базы создана миграция:

- `bakery/migrations/0003_order_user_alter_order_customer.py`.

Поле `customer` оставлено необязательным, чтобы сохранить совместимость с прежней моделью покупателей.

### 4. Авторизация, регистрация и выход

Добавлены маршруты:

- `/login/` - авторизация;
- `/register/` - регистрация;
- `/logout/` - выход из аккаунта.

Регистрация реализована через `RegistrationForm` в `bakery/forms.py` и `RegisterView` в `bakery/views.py`.

Созданы шаблоны:

- `bakery/templates/bakery/login.html`;
- `bakery/templates/bakery/registration.html`.

В `bakery_project/settings.py` добавлены настройки:

```python
LOGIN_URL = 'login'
LOGIN_REDIRECT_URL = 'home'
LOGOUT_REDIRECT_URL = 'home'
```

### 5. Разграничение прав доступа

Для операций создания, изменения и удаления справочных сущностей добавлен `StaffRequiredMixin`.

Обычный пользователь может:

- просматривать каталог;
- добавлять товары в корзину;
- оформлять заказ после авторизации;
- добавлять отзыв после авторизации.

Staff-пользователь может:

- создавать, редактировать и удалять товары;
- создавать, редактировать и удалять категории;
- создавать, редактировать и удалять производителей;
- создавать, редактировать и удалять акции;
- редактировать и удалять отзывы.

В шаблонах элементы управления скрываются в зависимости от прав:

- кнопки добавления сущностей выводятся только при `user.is_staff`;
- кнопки `Изменить` и `Удалить` выводятся только при `user.is_staff`;
- кнопка добавления отзыва выводится только авторизованным пользователям.

### 6. Кнопки в хедере

В `bakery/templates/bakery/base.html` добавлена проверка авторизации:

- если пользователь авторизован, отображается только кнопка `Выход`;
- если пользователь не авторизован, отображаются кнопки `Авторизация` и `Регистрация`.
